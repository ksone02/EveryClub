# EveryClub

## 프로젝트 규칙
- 코드, 폴더 구조는 김영한 강의 베이스로 작성하기
- Commit 규칙 준수하기
- Commit은 따로 Branch 생성 / Fork 후 PR하기
- 커밋 메시지는 "[Type] message" 형식으로 작성하기
  | **Type** | **Meaning** |
  | -------- | ------|
  | Feat    | 새로운 기능 |
  | Fix     | 버그 수정 |
  | Resolve | 충돌 해결 |
  | Build   | 빌드 관련 파일 수정 |
  | Chore   | 그 외 자잘한 수정 |
  | Style   | 스타일 변경 |
  | Docs    | 문서 수정 |
  | Refactor| 코드 리팩토링 |
  | Test    | 테스트 코드 수정 |
  | Add     | 새로운 라이브러리 / 파일 생성 |
